Sim2Sim granular robotics benchmark

Online material belief for robot excavation

A reproducible synthetic benchmark for estimating granular material family and Mohr-Coulomb-style properties from contact-rich robot interaction, then using that posterior to change excavation behavior under force and transport constraints.

Scope: this project currently claims synthetic Sim2Sim robustness and scoped control usefulness, not full Real2Sim deployment. The evidence chain is explicit: first estimate a material belief, then test whether that belief changes a downstream excavation action, then add scoped DDBot and real-force sanity checks.

Method

The core idea is to keep the granular medium inspectable: a blade/scraper interaction produces force and surface-change evidence, the model turns that evidence into an online material posterior, and the controller uses the posterior to choose a safer excavation action.

Generate paired materials

Sample sand, soil, gravel, and crunching debris with overlapping property ranges and nuisance actions.

Estimate online belief

Fuse MPM visual deformation summaries with wrench and end-effector kinematics frame by frame.

Validate Sim2Sim

Run GT and estimated materials through the same property-revealing bulldozing wedge task.

Close the loop

Use the posterior to select a trajectory that balances trench depth, transport, spillage, and force risk.

Estimator Stress Benchmark

The headline run uses held-out actions, held-out materials, overlapping property ranges, sensor noise, and vision corruptions. These gates are a readiness check for the posterior, not a real-world success claim by themselves.

Posterior-Conditioned Excavation

The main downstream test keeps the same GT MPM sand bed and the same blade interaction, then changes only the controller belief: no posterior, wrong posterior, estimated posterior, and GT property reference.

Estimated posterior 45.6 mm 0.65 2257 N 0 N 180.3
GT property reference 39.1 mm 0.56 2253 N 0 N 175.8

The GT property row is a controller reference under the same finite trajectory budget, not a mathematical optimum. When the estimated row is slightly stronger, the claim is not that the estimate is better than GT; it means the posterior was good enough to pick a GT-level action from the sampled controller candidates.

Scoped Comparison And Real-Force Checks

These checks are deliberately bounded. They support the claim that force-aware posterior control is useful, while avoiding a blanket claim that the full official DDBot system or real-world excavation has been solved.

Mini Proposal: Construction-Site Granular Digital Twin

The next research target is a site-level digital twin for sand, soil, and aggregate handling: estimate material belief from machine interaction, update the terrain twin, and replan excavation before overload or spillage happens.

Sense the site

Fuse RGB-D or LiDAR height maps with machine pose, bucket trajectory, hydraulic pressure, or F/T proxies.

Update material belief

Maintain an online posterior over density, friction, interface friction, cohesion, and moisture-like proxies.

Roll out the twin

Use a fast height-field or MPM surrogate to predict terrain change, reaction force, work, and overflow risk.

Optimize the next action

Choose the next bucket/tool motion against target shape, force limits, cycle time, and spillage constraints.

Proposed contribution: a construction digital twin that does not treat sand as a fixed visual surface. It continuously updates material uncertainty from interaction and uses that uncertainty in planning.
